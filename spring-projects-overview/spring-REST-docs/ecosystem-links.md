# Spring REST Docs — Ecosystem Links

Spring REST Docs is a **complementary tool** within the Spring ecosystem.  
It **does not replace Spring MVC, Spring Boot, or testing frameworks**, but integrates with them to provide **accurate, test-driven API documentation**.

---

## Core Spring Integrations

### 1. Spring Boot
- Simplifies **test and endpoint configuration**
- Allows documentation generation in the **context of a full application**

### 2. Spring MVC / WebFlux
- Defines the **REST endpoints** to be documented
- Supports tests with **MockMvc** or **WebTestClient**

### 3. Spring Security
- Enables documentation of **secured endpoints**
- Supports snippets for **authentication, headers, and roles**

---

## Testing Frameworks

- **JUnit 5 / 4** → Compatible test frameworks
- **Spock** → Groovy-based tests
- **Spring Test / MockMvc / WebTestClient** → Capture requests and responses during tests

---

## Documentation Tools

- **Asciidoctor** → Render snippets into **HTML, PDF, or other formats**
- **Markdown** → Lightweight alternative for repositories or GitHub
- **Spring Auto REST Docs (optional)** → Automates snippet generation with minimal manual code

## Additional Integrations / Notes

- Can be integrated with **CI/CD pipelines** for automatic documentation generation
- Works well with **microservices**, ensuring consistent docs across multiple services
- Does **not replace Swagger/OpenAPI**, but can be used **alongside it** for production-grade documentation
